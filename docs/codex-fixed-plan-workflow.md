# Codex 固定枠ワークフロー

このメモは、従量課金の生成 AI API をできるだけ使わず、ローカルの Codex 実行環境を使って記事要約とスライド構成作成を行うための設計方針です。

## 前提

このプロジェクトで想定するのは、Slack bot が OpenAI API やその他の従量課金 LLM API を直接呼び出す方式ではありません。

Slack bot は URL を受け付けてジョブとして保存します。その後、ローカル環境で動く Codex が未処理ジョブを読み取り、要約とスライド構成作成を実行します。

つまり、生成 AI の知的作業はローカルの Codex 利用枠で行い、Slack API と Google APIs は通知・保存などの外部連携に限定します。

## 推奨フロー

1. ユーザーが Slack に URL を投稿する
2. Slack bot が URL、投稿者、投稿チャンネルをジョブとして保存する
3. ローカルの Codex runner が未処理ジョブを検出する
4. Codex が記事や論文の内容を確認する
5. Codex が Google Slides 用のスライドアウトラインを作成する
6. Google Slides 作成スクリプトがアウトラインからデッキを生成する
7. Google Drive にデッキを保存する
8. Slack bot が完了通知とスライド URL を投稿する

## 従量課金 API を避けられる理由

- 要約やスライド構成作成の知的作業をローカル Codex の利用枠で行う
- Slack API と Google APIs は連携用 API であり、生成 AI のトークン課金 API ではない
- デフォルト構成では OpenAI API キーを必須にしない
- Slack から受けた URL は直接 LLM API に送らず、ジョブファイルとしてローカル処理に渡す

## 推奨する実装パターン

### ローカル runner 方式

Codex を利用できるローカル環境で、未処理ジョブを順番に処理します。

このプロジェクトではこの方式を本命にします。

想定する流れは次の通りです。

1. Slack bot が `jobs/pending/` にジョブファイルを作成する
2. ローカル runner が `jobs/pending/` を監視する
3. runner が Codex に「このジョブを処理して、スライドアウトライン JSON を作成して」と依頼する
4. Codex が `jobs/completed/` にアウトラインや実行結果を書き出す
5. Google Slides 作成処理が完了ジョブを読み取り、デッキを作る
6. Slack bot が完了通知を送る

## 初期検証パターン

完全自動化の前に、次のような半自動フローで検証すると安全です。

1. Slack bot は URL をジョブファイルとして保存する
2. ユーザーがローカル Codex に「未処理ジョブを処理して」と依頼する
3. Codex がアウトラインを作成する
4. スクリプトで Google Slides を生成する
5. Slack に完了通知する

この方式なら、従量課金 LLM API を使わずに、処理品質やスライド構成を確認しながら自動化できます。

## 採用しない方式

### Codex cloud / GitHub 委譲方式

Slack bot が GitHub issue、branch、または job file を作成し、Codex cloud に処理を委譲する方式は、このプロジェクトの主方針から外します。

理由は、今回の目的が「ローカルの Codex 実行環境を使い、固定課金枠またはプラン内利用枠を活用すること」だからです。

Codex cloud 自体が必ず従量課金 API になるという意味ではありませんが、ローカル Codex 前提の運用とはコスト管理や実行経路が変わります。そのため、ここでは採用候補から外します。

## トレードオフ

- 完全な API-to-API のリアルタイム要約サービスとは異なる
- ローカル runner を動かしておく必要がある
- Codex の利用上限やクレジットは契約プランに依存する
- 大量の URL を連続処理する用途では、キュー、優先度、失敗時の再実行設計が必要になる

## このプロジェクトでの扱い

このプロジェクトでは `SUMMARY_PROVIDER=codex_job` をデフォルトにしています。

初期状態では従量課金の LLM API を呼び出さず、Slack から受け付けた URL をローカル Codex が処理しやすいジョブとして扱います。
