# テンプレート方式による社内向けスライド生成

社内活用できる見た目にするには、Google Slides のテンプレートを先に用意し、Codex は記事内容をテンプレートの型に割り当てる方式が安定します。

## 基本方針

毎回 Codex にデザインを考えさせるのではなく、デザインは Google Slides テンプレートとして固定します。

Codex が担当するのは次の部分です。

- 記事や論文の要点抽出
- 社内向けの示唆への変換
- どのスライドにどのレイアウトを使うかの判断
- テンプレートに流し込む JSON の生成

Google Slides API が担当するのは次の部分です。

- テンプレートの複製
- プレースホルダーの置換
- 必要なスライドの複製・削除
- Google Drive への保存

## 推奨テンプレート構成

最初は次の 8 種類のレイアウトを用意すると使いやすいです。

1. 表紙
2. 章扉
3. キーメッセージ
4. 3 点要約
5. 比較表
6. プロセス図
7. リスク・機会
8. まとめ・次アクション

## プレースホルダー規約

テンプレート内の差し替え箇所は、次のような文字列にします。

```text
{{DECK_TITLE}}
{{DECK_SUBTITLE}}
{{SOURCE_URL}}
{{GENERATED_DATE}}
{{SLIDE_TITLE}}
{{BODY_1}}
{{BODY_2}}
{{BODY_3}}
{{FOOTER}}
```

レイアウトごとに必要なプレースホルダーを固定しておくと、Google Slides API の `replaceAllText` で安定して差し替えられます。

## 中間 JSON

Codex が生成するアウトラインは、単なる箇条書きではなく、レイアウト種別を含む中間 JSON にします。

```json
{
  "title": "Quantum AI の優位性をどう引き出すか",
  "subtitle": "社内向け技術調査メモ",
  "slides": [
    {
      "layout": "three_takeaways",
      "title": "重要ポイント",
      "bullets": [
        "古典データの量子ロード問題に新しい道筋を示す",
        "メモリ効率の面で量子優位性を主張している",
        "生成 AI への即時適用ではなく、基礎 ML タスクが主対象"
      ]
    }
  ]
}
```

## 社内活用向けの変換ルール

記事要約をそのままスライド化するより、次の観点を足すと社内で使いやすくなります。

- 何が新しいのか
- 自社に関係する可能性はどこか
- 今すぐ使える話か、将来注視すべき話か
- 技術的な不確実性は何か
- 次に読むべき論文・試すべき検証は何か

## 実装ステップ

1. 社内テンプレートの Google Slides を作成する
2. テンプレートの presentation ID を `.env` の `GOOGLE_SLIDES_TEMPLATE_ID` に設定する
3. 各レイアウトにプレースホルダーを配置する
4. Codex が `layout` 付きアウトライン JSON を生成する
5. Google Slides API でテンプレートを複製する
6. レイアウトごとにスライドを複製し、プレースホルダーを置換する

## 固定課金枠での扱い

この方式では、要約、構成判断、社内向け示唆の作成はローカル Codex が行います。

従量課金の LLM API は使わず、Google Slides API はテンプレート複製と文字差し替えだけに使います。
