# ステップ2：ネーム詳細化（見開き漫画対応・キャラID呼び出し方式）

ステップ1で作成した「確定ネーム骨子一覧（マスター）」「キャラID属性ブロック」を元に、各見開きスライドの各コマを実制作レベルまで詳細化してください。

## 【最重要ルール：1回で完走】
- 全見開きを1回のチャット応答で出力する
- 分割出力・バッチ出力・「続きを出力してよいか」のような確認の問い返しはしない
- 文字数が多くなっても、中断せず最後まで出力を続ける
- もし応答途中で打ち切られた場合でも、次のバッチに分ける対応はしない（ユーザーが「続き」と言ったら、打ち切られた箇所の途中から出力を再開するだけで、最初からやり直さない）

## 【文字数を抑えるための記述圧縮ルール】
- キャラ属性はステップ1のキャラID属性ブロックで定義済み。コマごとには書かない
- 各コマではキャラID（C1, C2等）で呼び出し、「そのコマでの差分」だけを書く
- 1項目1行の簡潔形式で記述
- 「変化なし」「通常」等の当たり前の情報は省略してよい

## 【最重要ルール（内容面）】
- ステップ1の見開き数・各スライドのコマ数・テンプレートID・読み順は変更しない
- コマの追加・削除・統合・分割は禁止
- 座標は1920×1080pxの範囲内、単位はpx
- 読み順：右ページ→左ページ、各ページ内は上→下、同段は右→左
- キャラを登場させるコマには必ずキャラIDを明示
- **セリフは各キャラのキャラID属性ブロックに記載された口調（一人称・語尾・口癖）を厳守する。性別・性格と一致した口調で書く**

## 【画風パターン参照】
ソースに読み込まれている画風ファイルの指定に従う。1つだけ有効化する。未指定なら画風Aデフォルト。

## 【各コマへの追加項目】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 1. 【座標】(X_start, Y_start) - (X_end, Y_end)
ステップ1のテンプレート定義から転記。1920×1080 相当のスライド内の座標。

### 2. 【コマ枠】
タイプ（通常矩形／斜めカット／ぶち抜き／多角形）／線色・太さ（画風準拠なら「画風A準拠」等で省略可）

### 3. 【構図】
カメラ位置・アングル（1行で簡潔に）

### 4. 【キャラ差分】★IDで呼び出し、差分のみ★
```
キャラ：C1
  位置：右／ポーズ：仁王立ち／表情：決意／視線：前方／画面比率：バストアップ
キャラ：C2
  位置：左寄り／ポーズ：うつむく／表情：困惑／視線：下／画面比率：全身
```
（1キャラ1〜2行。キャラIDのみで呼び出す。キャラシート画像の参照はステップ3でSlide単位で行うため、ここにはファイル名を入れない）

### 5. 【背景】
場所・時間帯・雰囲気を1〜2行で

### 6. 【吹き出し】
```
B1 右上 帯ナレーション サイズ中
B2 左中 通常楕円 サイズ中 しっぽ→C1
```
（1吹き出し1行）

### 7. 【セリフ】
```
B1 [ナレーション]「あの日、世界が変わった——」
B2 [C1]「行くぞ！」
```
（キャラIDで指定し、そのキャラの口調で書く）

### 8. 【効果音】
オノマトペがあれば：内容・配置・サイズ・傾き・色を1行で。無ければ省略。

### 9. 【視線誘導】
次コマへの流れを1行で。明白なら省略可。

### 10. 【エフェクト】
集中線／ベタフラッシュ／汗マーク等あれば1行で。無ければ省略。

### 11. 【モノクロ演出】（任意・基本不要）
原則コマはフルカラーで描画される。
ただし回想・緊迫・衝撃・死・絶望・時間停止など、**意図的にモノクロ（グレースケール）で演出したい特定のコマ**のみ、以下を1行で明記する：
```
モノクロ演出：有効（理由：回想シーン／緊迫・絶望／衝撃の瞬間 など）
```
この指定がないコマは必ずフルカラーで描画される。
モノクロ演出は多用しない。1作品あたり数コマ程度が目安。全編モノクロにする場合は、ユーザーがステップ3の冒頭カラー原則指示を差し替える運用とする。

**【フォーマットルール】**
各見開きスライドの見出し行には、ステップ1で決めた形式をそのまま引き継ぐ：
```
Slide N｜manga spread page, two-page layout, right-to-left reading order｜Txx
```
見開き指定を含むこの見出しで、画像生成AIが見開きレイアウトとして解釈する。

  Panel 1
    座標：(0, 0) - (1920, 1080)
    コマ枠：画風A準拠
    構図：C1のバストアップ、煽りアングル、見開き全面
    キャラ：C1
      位置：中央／ポーズ：仁王立ち／表情：決意／視線：左下／画面比率：バストアップ
    背景：朝の街、水色グラデーション、遠景に街並み
    吹き出し：B1 右上 帯ナレーション サイズ中
    セリフ：B1 [ナレーション]「あの日、世界が変わった——」
    効果音：なし
    視線誘導：右ナレーション→中央C1→次見開きへ

## 【自己点検】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 全コマに必要な項目が埋まっているか
- 座標がテンプレート定義と一致しているか
- キャラ登場コマでキャラIDが明示されているか
- **セリフが各キャラの性別・性格・口調と一致しているか**
- 吹き出しの読み順が右→左→下を守っているか（見開きなので右ページが先）
- 指定された総見開き数と完全一致しているか
- **モノクロ演出指定は本当に必要なコマのみに限定しているか（全編モノクロは避ける）**

## 【このステップの出力】
- 上記圧縮形式の【詳細ネーム一覧（マスター・確定版）】を、**1回のチャット応答で全見開き分出力する**
- 分割・中断・問い返しは禁止
- 出力完了後、ユーザーに「ステップ3に進んでよいか」を確認する

## 【もし応答が途中で打ち切られた場合の復旧方法】
ユーザーが「続き」と送ってきた場合：
- 絶対に最初からやり直さない
- 打ち切られた箇所（最後に出力したSlide X / Panel Y）を特定し、**その直後から**出力を再開する
- 「Slide Xの途中から再開します」と明言してから続きを書く

【重要】
この出力はステップ3で user_steering_prompt に丸ごと転記される。
ステップ1の【キャラID属性ブロック】もステップ3で同時に転記されるため、画像生成AIはキャラIDだけで固有属性・性別・英語タグに完全アクセスできる。
**キャラシート画像の参照はステップ3でSlide見出し直後に@記法で行う。コマ内にファイル名は入れない。**
